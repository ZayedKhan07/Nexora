# Nexora 🏥

Nexora is a modern healthcare communication and workflow platform designed to bridge the gap between clinics and patients.

It provides a unified system for managing patient interactions, appointments, intake, and follow-ups — all in one place.

---

## 🚀 Vision

To become the operating system for patient communication in healthcare.

Nexora is not just a booking tool — it is a patient journey platform that enhances efficiency for clinics and experience for patients.

---

## 🧩 Core Concept

Nexora is a dual-role platform:

- 👤 **Patients** → manage appointments, forms, and follow-ups
- 🏥 **Healthcare Providers** → manage clinic workflow, patients, and operations

Users select their role at login and are routed to different interfaces.

---

## 🔑 MVP Scope (Phase 1)

### Provider (Clinic Dashboard)
- Dashboard (daily overview)
- Patient management
- Appointment management
- Intake form builder
- Form responses tracking
- Manual reminders/tasks
- Basic analytics

### Patient App
- Account creation (select clinic)
- View & manage appointments
- Fill intake forms
- Basic reminders
- Profile management

---

## 🔗 Patient ↔ Clinic Model

- Patients can select their healthcare provider during signup
- Each patient is linked to a clinic
- Future: support for multiple clinic connections

## 🤖 Development Approach

This project follows an AI-assisted development workflow:

1. Break features into small tasks
2. Define each task in `/prompts`
3. Use Gemini CLI to generate code
4. Test and validate
5. Commit changes to GitHub

---

## 📌 Development Rules

- Build in small, testable tasks
- No large, undefined feature builds
- Keep components modular and reusable
- Avoid hardcoding values
- Maintain clean folder structure
- Commit after every working feature
